# Functions

Functions in Slynx can be written in multiple styles, as seen before, but the thing is that executable code can only exist inside them. Thus no global state is avaible(yet). It's idealized to have, but, restrict.

### Variables

Variable creation follows Rust syntax:

```slynx
let a = 5;       // immutable
let mut b = 12;    // mutable
```

Note: Mutability checks are not implemented yet, so `mut` is accepted but not enforced.

### Conditionals

Conditionals work like Rust and can return a value:

```slynx
if a > b {
  a
} else {
  b
}
```

### While Loops

While loops exist as in Rust, but only work for boolean conditions:

```slynx
while condition {
  // loop body
}
```

### Expressions

Slynx supports various binary expressions:

- **Arithmetic**: `+`, `-`, `*`, `/`
- **Bitwise**: `<<`, `>>`, `|`, `&`, `^`
- **Boolean**: `&&`, `||`, `==`, `!=`, `<`, `>`, `<=`, `>=`

Note: Unary expressions are not yet implemented.

## Returns

Slynx is strongly typed — the function signature **must** declare exactly what type is returned. There is no type inference for return types.

```slynx
func add(a: int, b: int): int -> a + b;
```

### Tuples

Tuples are supported with parentheses:

```slynx
func get_pair(): (int, str, float) -> (1, "hello", 3.14);
```

Field access is not yet implemented, but being developed.
